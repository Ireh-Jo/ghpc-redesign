# 디자이너 협업 워크플로 — LOCKED: 이레 주도 + 디자이너 비주얼·리뷰

## 목적

디자이너 박정민·장진경과의 협업 방식을 확정한다. 안건 2-2(디자이너 협업 방식)의 잠금 결정.

## 현 상태 (사실관계 — 2026-05-19 팀 확인)

- 디자이너 2명: 박정민, 장진경
- 사용 가능 도구: **일러스트레이터, 포토샵** 만
- Figma·웹플로우 **사용 경험 0** — 새 도구 학습은 일정상 불가
- 박정민: "만들어진 템플릿에 이미지 변경하는 정도의 간단한 웹사이트 가능". 7~8월 교회 여름행사로 가용 시간 적음. 출근일 화·수·목·일.
- 장진경: 메인 디자인을 단독으로 잡기엔 자신 없음. 협업 의지는 있음.
- 결론: "Figma 시안" 및 "디자이너 메인 디자인 주도" 두 가정 모두 불가.

## 잠금 결정 (2026-05-19)

**이레가 디자인 시스템·시안·구현을 주도하고, 디자이너는 비주얼 자산 제작 + 시안 리뷰를 담당한다.**

| 역할 | 담당 | 산출물 |
|---|---|---|
| 디자인 토큰·IA·시안 구조 | 이레 | `context/design/*`, `prototypes/designs/*.html` |
| 시안 HTML 제작 | 이레 (AI 도구 활용) | `prototypes/designs/*.html` |
| 비주얼 자산 (사진·일러스트·아이콘 일부) | 박정민·장진경 | PNG / SVG / JPG 파일 |
| 시안 리뷰·피드백 | 박정민·장진경 | 브라우저로 확인 → 코멘트 |
| 본 개발 구현 | 이레 | `app/`, `components/` |

→ 안건 2-2의 "디자인만 vs 디자인+퍼블리싱" 질문에 대한 답: **디자이너는 퍼블리싱 없음. 비주얼 자산 + 리뷰만.**

## Figma의 빈자리 — 시안 HTML이 디자인 산출물

Figma를 안 쓰므로 디자인 합의의 단일 표면은 **시안 HTML**(`prototypes/designs/*.html`)이다.

- 7월 안건의 "Figma 시안"은 실질적으로 **"시안 HTML + 부분 비주얼 모킹"** 으로 대체
- 시안 HTML이 곧 본 개발의 시각 스펙 역할 (단, 코드 직접 복붙 금지 — `CLAUDE.md` 절대 규칙 5)
- 현재 `prototypes/designs/index.html`(메인) 1개 → 서브 페이지 시안 추가 제작 가치가 커짐

## 워크플로

```
1. 이레: 시안 HTML 제작 (Cursor / v0 / Claude Code 활용)
2. 디자이너: 브라우저로 시안 확인 → 피드백
3. 이레: 피드백 반영 → 시안 동결
4. 디자이너: 필요한 비주얼 자산을 일러/포샵으로 제작 (사진 보정·일러스트·아이콘)
5. 이레: 자산을 본 개발 코드에 통합 (최적화·접근성 재검증)
```

## 비주얼 자산 전달 규칙

- 사진: JPG, 가로 2000px 이상 원본. 보정본 전달.
- 일러스트·아이콘: SVG 우선, 불가 시 PNG(투명 배경, 2x 해상도)
- 색상: `context/design/01-color.md` 토큰 범위 내에서 작업 (임의 색 추가 시 이레와 협의)
- 파일명: `{페이지}-{용도}-{번호}` (예: `main-hero-01.jpg`)
- 전달 경로: > DECISION NEEDED: 공유 폴더 위치 (Google Drive / 교회 미디어 서버 등) 합의 필요

## 시안 HTML 리뷰 절차

1. 이레가 시안 HTML 링크(또는 파일) 공유
2. 디자이너는 브라우저에서 확인 — 모바일·데스크톱 둘 다
3. 피드백은 화면 캡처에 주석 또는 텍스트로 (Figma 코멘트 불가하므로)
4. 이레가 반영 여부·사유 회신 → 합의 시 시안 동결

## 일정 리스크

- 박정민 7~8월 가용 시간 적음 → 비주얼 자산 요청은 6~7월 초에 미리, 8월 의존 최소화
- 디자이너 리뷰 1~2일 텀 가정하고 일정 버퍼 확보
- 디자이너 병목 시: AI 도구로 임시 비주얼(스톡 이미지·생성 일러스트) 후 추후 교체
