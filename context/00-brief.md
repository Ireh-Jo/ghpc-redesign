# 00. 프로젝트 브리프

## 무엇을

경향교회 홈페이지 **본 개발**. Next.js 14(App Router) + Supabase + Vercel.
시안 단계 종료(2026-05). 11월 말 ~ 12월 초 오픈 목표.

## 누구를 위해

- **새신자** (교회 처음 방문/처음 검색하는 사람) — 진입 친화도 최우선
- **3040 세대** — 모바일 우선, 빠른 정보 탐색, 현대적 UI 기대치
- **미디어팀 2명** (콘텐츠 운영) — 어드민 UX의 직접 사용자
- **이레** (개발·기술운영) — 장기 유지보수 부담 최소화가 설계 기준

## 왜 개편하는가

기존 6개 카테고리(소개·예배·교육기관·연합기관·소식·e교회행정) → **4개 + 메인 + 새가족 강조 메뉴**로 단순화.
이유: 새신자 진입 동선이 흩어져 있고, 카테고리명이 내부 조직 기준이라 외부에서 이해 어려움.

## 핵심 컨셉 (기획서 원문)

1. **새신자 및 3040 세대 친화적**
2. **목차 재조정** (6→4개 + 새가족 별도)
3. **최신 트렌드 반영** — 스크롤 다운 중심 · 영상 활용 · 드롭다운 메뉴

## 비-목표 (1차 오픈 범위 밖)

- AI Q&A 기능 (별도 프로젝트로 분리)
- 회원 시스템·로그인 (회원정보 별도 관리 없는 것으로 이해 — 확정 필요)
- 결제·온라인 헌금 처리 (기존 채널 링크만)
- e-교회행정 시스템 자체 개발 (외부 링크만)

## 성공 기준

- 새신자가 메인에서 3클릭 안에 "주일예배 시간"과 "오시는 길" 도달
- 모바일 360px에서 가로 스크롤 없이 모든 기능 사용 가능
- Core Web Vitals: LCP ≤ 2.5s, CLS ≤ 0.1, INP ≤ 200ms (`guardrails/05-performance.md`)
- 미디어팀 2명이 별도 교육 없이 매주 콘텐츠 업로드 가능
- SSL·DB·배포가 1년간 사람 손 안 대고 유지 (`context/ops/*`)
