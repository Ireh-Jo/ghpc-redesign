# 00. DO / DON'T — 절대 규칙

> 야생마(AI)에게 마구(harness)를 채우는 룰. 어기면 시안 폐기.

## DO — 무조건 한다

1. **시안 시작 전 `context/` 5개 문서를 모두 한 번 읽는다.** 한 번도 안 읽었으면 짐작으로 코드 쓰지 말 것.
2. **컬러/타이포/간격은 `context/01-design-system.md`의 토큰만 사용한다.** 임의 HEX·임의 px 금지.
3. **5개 페이지 모두 같은 헤더·푸터·컨테이너 너비를 쓴다.** 변경 시 5개 모두 동시 업데이트.
4. **콘텐츠 텍스트는 `context/03-content-inventory.md`에서 가져온다.** 없으면 placeholder임을 명시(`[목사님 인사말]`).
5. **모바일 360px에서 가로 스크롤 없어야 한다.** 새 섹션 만들 때마다 모바일부터 확인.
6. **이미지에는 alt 텍스트.** placeholder도 의미 있는 alt.
7. **결정사항 변경 시 → `context/` 문서 먼저 수정 → 시안 5개 일괄 반영.** 시안 코드만 고치고 문서 안 고치면 다음 시안에서 또 어긋남.
8. **Tailwind CDN으로만 작업.** 빌드 도구 도입 금지.
9. **모르면 placeholder + `> DECISION NEEDED` 코멘트.** 추측으로 박지 말 것.

## DON'T — 절대 하지 않는다

1. **임의 컬러 추가 금지.** 디자인시스템 토큰 외 색은 쓰지 않는다. "예뻐 보여서" 추가하지 말 것.
2. **이모지 사용 금지.** UI 아이콘은 lucide 또는 인라인 SVG만. (사용자가 명시 요청 시에만 예외)
3. **자동재생 캐러셀 금지.** 접근성·성능 저하. 카드 그리드로 대체.
4. **무거운 애니메이션 금지.** 패럴랙스·GSAP·Lottie 풀스크린 모두 금지. 페이드인·호버만.
5. **헤더 통일 깨기 금지.** 5개 페이지 헤더는 한 글자도 다르면 안 된다 (현재 페이지 active 표시는 OK).
6. **사용자 사진 임의 합성 금지.** 목사님·교인 얼굴 Unsplash·AI 생성 금지. 무조건 실루엣/추상 placeholder.
7. **신학적 단언 금지.** 인사말·신학노선 텍스트는 placeholder임을 항상 명시. 임의로 신앙고백 문구 만들지 말 것.
8. **카테고리 추가 금지.** GNB는 5개(교회소개·예배와교육·목양과사역·교회활동·새가족)만. 기획서가 정한 핵심.
9. **시안 5개 한꺼번에 만들지 않는다.** 메인 → 통과 → 나머지 4개. 메인 미통과 상태에서 서브 만들면 무조건 재작업.
10. **버전 차이 두기 금지.** "메인 시안 A안" 같은 다양화는 사용자 요청 전엔 하지 않는다. 시안 5개는 다른 페이지지, 다른 스타일이 아니다.

## 작업 중단 트리거

다음 상황이면 즉시 멈추고 사용자에게 묻는다:

- `context/01-design-system.md`에 `DECISION NEEDED` 라인이 남아있는데 시안 시작하라는 지시를 받았을 때
- 콘텐츠가 인벤토리에 없는데 사용자가 "그냥 적당히 채워줘"라고 했을 때 (신학적 단언 위험)
- 5번째 시안에서 1번째와 토큰이 다르다는 걸 발견했을 때 (1번째 기준으로 5개 모두 재정합)
